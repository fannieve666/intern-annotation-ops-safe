# 发任务示例

下面是一份适合公开展示的脱敏版任务通知示例。

```text
【多维度音乐标注任务通知】

各位同学好，

本周任务已安排完毕，请以下同学登录标注平台，确认是否可以看到对应的任务界面：
@标注员01 @标注员03 @标注员05

如可以看到，即表示已成功领取任务，可以开始标注。

【任务说明】
一、本次任务共 20 条/人，请在 2026-05-03 18:00 前完成提交。
二、标注时请同时打开以下文档：
1）《多维度打分标准》：https://demo.example.com/standards
2）问题提报文档：https://demo.example.com/issues
三、提交格式为 CSV，请保持字段顺序一致。

【标注维度】
音质 / 演唱 / 悦耳度 / 记忆性 / 结构性
分值范围为 1.0～5.0，步长为 0.5。
每个维度都需要选择对应评论标签；若出现混合极性，请注明主导判断。

感谢大家的配合。
CC：@项目负责人
```

## 这份示例体现了什么

- 输出语气偏业务协作，不会过硬
- 文案适合放到群通知、飞书或企业微信中
- 保留了任务管理所需的关键字段，但不暴露真实业务信息
